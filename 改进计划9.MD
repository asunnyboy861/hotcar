# HotCar 改进计划 9 - 数据流优化与用户体验增强

> **版本**: 1.0  
> **日期**: 2026-03-13  
> **目标**: 优化设置数据持久化、增强错误处理、完善空状态引导  
> **影响范围**: Settings, Weather, Statistics, Onboarding 模块

---

## 📋 执行摘要

基于对 HotCar 软件用户使用流程和数据流的深度分析，本改进计划针对以下核心问题进行优化：

1. **设置数据持久化问题**: 温度单位自动检测会覆盖用户手动设置
2. **设置保存性能问题**: 每次属性变更都同步写入 UserDefaults，缺少防抖
3. **统计数据一致性**: sessions 和 statistics 存在数据不一致风险
4. **错误处理不足**: 天气 API 失败、位置权限被拒绝等场景缺少降级方案
5. **空状态缺失**: 统计数据为空时缺少引导性提示
6. **数据导出功能**: 有方法但缺少 UI 入口

## 🎯 第一部分：需求描述

### 1.1 设置数据持久化优化

**问题描述**:
```swift
// AppSettings init 中
init() {
    let locale = Locale.current
    let regionCode = locale.region?.identifier ?? "US"
    let fahrenheitRegions = ["US", "LR", "MM"]
    self.temperatureUnit = fahrenheitRegions.contains(regionCode) ? .fahrenheit : .celsius
}
```

**问题影响**:
- 用户手动修改温度单位后，下次启动应用会被重置
- 用户体验差，设置无法持久保存

**需求**:
- 添加 `isUserOverridden` 标记，区分用户手动设置和自动检测
- 首次启动使用自动检测，后续优先使用用户保存的设置
- 用户手动修改后，标记为已覆盖，不再自动重置

### 1.2 设置保存防抖优化

**问题描述**:
```swift
// SettingsViewModel 中
@Published var temperatureUnit: AppSettings.TemperatureUnit {
    didSet {
        settings.temperatureUnit = temperatureUnit
        saveSettings()  // 每次变更都同步写入
    }
}
```

**问题影响**:
- 快速切换设置（如连续点击 Stepper）会导致频繁写入 UserDefaults
- 可能影响性能，虽然 UserDefaults 写入是异步的，但仍存在优化空间

**需求**:
- 使用 Combine 的 `debounce` 操作符实现防抖
- 延迟 0.5 秒后保存，避免频繁写入
- 应用退出时立即保存，确保数据不丢失

### 1.3 统计数据一致性优化

**问题描述**:
```swift
// StatisticsService
@Published var sessions: [WarmUpSession] = []
@Published var statistics: WarmUpStatistics = WarmUpStatistics()
```

**问题影响**:
- `statistics` 是基于 `sessions` 计算的，但两者独立发布
- 直接修改 `sessions` 不会自动触发 `statistics` 更新
- 容易遗漏 `calculateStatistics()` 调用

**需求**:
- 使用 Combine 将 `statistics` 绑定到 `sessions`
- 任何 `sessions` 变更自动触发 `statistics` 重算
- 移除手动调用 `calculateStatistics()` 的需求

### 1.4 天气服务错误处理增强

**问题描述**:
```swift
// WeatherService
func fetchCurrentTemperature() async {
    // ...
} catch {
    await MainActor.run {
        self.errorMessage = error.localizedDescription
        self.isLoading = false
    }
}
```

**问题影响**:
- API 失败时只设置 `errorMessage`，没有降级方案
- 用户看不到错误提示，界面显示空白或加载中
- 位置权限被拒绝后没有引导用户开启

**需求**:
- 添加错误状态枚举，区分网络错误、权限错误、服务器错误
- API 失败时使用缓存数据或默认值
- 添加位置权限被拒绝的引导提示
- 提供手动刷新按钮和重试机制

### 1.5 统计数据空状态引导

**问题描述**:
- 新用户首次使用，统计数据为空
- 图表组件显示空白，用户不知道如何使用

**需求**:
- 添加空状态视图，显示引导性提示
- 提示用户"完成首次预热后将显示统计数据"
- 提供快速开始按钮，引导用户启动计时器

### 1.6 数据导出 UI 入口

**问题描述**:
- `SettingsViewModel` 中有 `exportData()` 方法
- 但 `SettingsView` 中只有 "Clear All Data"，没有 "Export Data" 按钮

**需求**:
- 在 Settings 页面添加 "Export My Data" 按钮
- 使用 `ShareSheet` 分享导出的 JSON 数据
- 支持导出到文件、邮件、消息等

```swift
struct AppSettings: Codable {
    // 现有属性...
    
    // MARK: - User Override Tracking
    
    /// 标记用户是否手动覆盖过温度单位设置
    var isTemperatureUnitOverridden: Bool = false
    
    // MARK: - Initialization with Region Detection
    
    init() {
        // 从 UserDefaults 加载保存的设置
        if let data = UserDefaults.standard.data(forKey: "com.hotcar.settings"),
           let savedSettings = try? JSONDecoder().decode(AppSettings.self, from: data) {
            // 使用保存的设置
            self = savedSettings
        } else {
            // 首次启动，使用地区检测
            self = AppSettings.createWithRegionDetection()
        }
    }
    
    /// 使用地区检测创建默认设置
    private static func createWithRegionDetection() -> AppSettings {
        var settings = AppSettings()
        let locale = Locale.current
        let regionCode = locale.region?.identifier ?? "US"
        let fahrenheitRegions = ["US", "LR", "MM"]
        settings.temperatureUnit = fahrenheitRegions.contains(regionCode) ? .fahrenheit : .celsius
        settings.isTemperatureUnitOverridden = false
        return settings
    }
}
```

### 2.2 设置保存防抖优化方案

#### 2.2.1 SettingsViewModel 防抖实现

**文件**: `Features/Settings/SettingsViewModel.swift`

**修改内容**:
```swift
@MainActor
final class SettingsViewModel: ObservableObject {
    
    // MARK: - Published Properties
    
    @Published var temperatureUnit: AppSettings.TemperatureUnit = .celsius
    @Published var defaultTimerDuration: Int = 15
    // ... 其他属性 ...
    
    // MARK: - Private Properties
    
    private var saveCancellable: AnyCancellable?
    private let saveSubject = PassthroughSubject<Void, Never>()
    
    // MARK: - Initialization
    
    init() {
        loadSettings()
        setupDebounce()
    }
    
    // MARK: - Setup
    
    private func setupDebounce() {
        saveCancellable = saveSubject
            .debounce(for: .milliseconds(500), scheduler: RunLoop.main)
            .sink { [weak self] in
                self?.performSave()
            }
    }
    
    // MARK: - Public Methods
    
    /// 触发保存（带防抖）
    func triggerSave() {
        saveSubject.send()
    }
    
    /// 立即保存（用于应用退出时）
    func saveImmediately() {
        saveCancellable?.cancel()
        performSave()
    }
    
    private func performSave() {
        settingsService.updateTemperatureUnit(temperatureUnit)
        settingsService.updateDefaultTimerDuration(defaultTimerDuration)
        // ... 其他设置项 ...
    }
}
```

**检验标准**:
- [ ] 连续快速修改设置只触发一次保存
- [ ] 停止操作 0.5 秒后自动保存
- [ ] 应用退出时调用 `saveImmediately()` 确保数据不丢失

```swift
@MainActor
final class StatisticsService: ObservableObject {
    
    // MARK: - Published Properties
    
    @Published var sessions: [WarmUpSession] = []
    
    /// 自动计算的统计数据，基于 sessions
    @Published private(set) var statistics: WarmUpStatistics = WarmUpStatistics()
    
    // MARK: - Private Properties
    
    private var cancellables = Set<AnyCancellable>()
    
    // MARK: - Initialization
    
    private init() {
        loadSessionsSync()
        setupBindings()
    }
    
    // MARK: - Combine Bindings
    
    private func setupBindings() {
        $sessions
            .receive(on: RunLoop.main)
            .sink { [weak self] sessions in
                self?.calculateStatistics(from: sessions)
            }
            .store(in: &cancellables)
    }
    
    // MARK: - Private Methods
    
    private func calculateStatistics(from sessions: [WarmUpSession]) {
        // 计算逻辑...
        statistics = WarmUpStatistics(
            totalSessions: sessions.count,
            totalWarmUpTime: sessions.reduce(0) { $0 + $1.duration },
            // ... 其他统计字段
        )
    }
    
    // 移除所有手动调用 calculateStatistics() 的地方
}
```

**检验标准**:
- [ ] 添加/删除/修改 session 后 statistics 自动更新
- [ ] 无需手动调用 calculateStatistics()
- [ ] 统计数据始终与 sessions 保持一致

```swift
// MARK: - Weather Error

enum WeatherError: Error, LocalizedError {
    case invalidURL
    case networkError(Error)
    case serverError(Int)
    case decodingError(Error)
    case locationPermissionDenied
    case noCachedData
    
    var errorDescription: String? {
        switch self {
        case .invalidURL:
            return "Invalid weather service URL"
        case .networkError:
            return "Network connection failed. Please check your internet connection."
        case .serverError(let code):
            return "Weather service unavailable (Error \(code)). Please try again later."
        case .decodingError:
            return "Failed to parse weather data"
        case .locationPermissionDenied:
            return "Location access is required for accurate weather data"
        case .noCachedData:
            return "No cached weather data available"
        }
    }
    
    var recoverySuggestion: String? {
        switch self {
        case .locationPermissionDenied:
            return "Please enable location access in Settings > Privacy > Location Services"
        case .networkError, .serverError:
            return "Pull down to refresh or check your connection"
        default:
            return nil
        }
    }
}
```

#### 2.4.2 WeatherService 错误处理增强

**文件**: `Features/WeatherIntegration/WeatherService.swift`

**修改内容**:
```swift
@MainActor
final class WeatherService: ObservableObject {
    
    // ... 现有属性 ...
    
    /// 当前错误状态
    @Published var currentError: WeatherError?
    
    /// 是否显示错误提示
    @Published var showError: Bool = false
    
    /// 缓存的温度数据
    private var cachedTemperature: Double?
    private let cacheKey = "com.hotcar.weather.cache"
    private let cacheTimestampKey = "com.hotcar.weather.cacheTimestamp"
    
    // MARK: - Public Methods
    
    func fetchCurrentTemperature() async {
        isLoading = true
        currentError = nil
        
        // 检查位置权限
        guard locationService.authorizationStatus == .authorizedWhenInUse ||
              locationService.authorizationStatus == .authorizedAlways else {
            handleError(.locationPermissionDenied)
            // 使用缓存数据作为降级
            useCachedDataOrDefault()
            return
        }
        
        // ... 现有请求逻辑 ...
        
        do {
            // ... 请求成功处理 ...
            await MainActor.run {
                self.currentTemperature = weatherData.currentWeather.temperature
                self.cacheTemperature(weatherData.currentWeather.temperature)
                self.lastUpdateTime = Date()
                self.isLoading = false
            }
        } catch {
            handleError(.networkError(error))
            // 使用缓存数据作为降级
            useCachedDataOrDefault()
        }
    }
    
    // MARK: - Private Methods
    
    private func handleError(_ error: WeatherError) {
        currentError = error
        showError = true
        isLoading = false
    }
    
    private func cacheTemperature(_ temperature: Double) {
        UserDefaults.standard.set(temperature, forKey: cacheKey)
        UserDefaults.standard.set(Date().timeIntervalSince1970, forKey: cacheTimestampKey)
        cachedTemperature = temperature
    }
    
    private func useCachedDataOrDefault() {
        // 尝试使用缓存数据
        if let cached = UserDefaults.standard.object(forKey: cacheKey) as? Double,
           let timestamp = UserDefaults.standard.object(forKey: cacheTimestampKey) as? TimeInterval {
            let cacheDate = Date(timeIntervalSince1970: timestamp)
            // 缓存 1 小时内有效
            if Date().timeIntervalSince(cacheDate) < 3600 {
                currentTemperature = cached
                return
            }
        }
        
        // 使用默认温度（0°C）
        currentTemperature = 0
        currentError = .noCachedData
    }
}
```
